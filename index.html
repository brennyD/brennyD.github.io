Part 1: Data Gathering¶

We will be using the New York Times API to get the bestseller data and the OpenLibrary API to get additional information on each book.

New York Times API - Data Extraction and Compilation¶

Our team used the New York Times Book API. This API provides information about book reviews and The New York Times Best Sellers lists.

According to the New York Time's website,

"The (bestseller) rankings reflect unit sales reported on a confidential basis by vendors offering a wide range of general interest titles published in the United States. Every week, thousands of diverse selling locations report their actual sales on hundreds of thousands of individual titles. The panel of reporting retailers is comprehensive and reflects sales in tens of thousands of stores of all sizes and demographics across the United States."

In order to get the bestseller data, there were several steps in the process

  1. Our team had to register an account with the New York Times to get a key to access the data.
  2. We created a function that returned a list of all the dates in 2018 in "YYYY-MM-DD" format. This is the format the Book API takes in date data.
  3. The New York Times has bestsellers in 61 different categories. In order to narrow down the data that we analyze, we chose the two most general categories from this list. Namely - Combined print and e-book fiction and Combined print and e-book non-fiction
  4. API calls were made for each day of 2018 for these two categories. All the data returned by the New York Times API is in the JSON format. This data was converted into a pandas dataframe and concatenated into one unified dataframe.
  5. According to the guidelines for using the New York Times API, you can only make one API call every 6 seconds. Thus, creating this dataframe took a total of 73 minutes.
  6. Once the dataframe was created, we were able to store it in the form of a CSV file. This CSV file is what we will be using for the rest of the analysis.

Since both of these datasets do not contain all of the data we need, we need to find another source of information to fill out details on these books. One such resource we are utilizing is Open Library, which is an open source library that catalogs the world's books. Therefore, we wrote a few helper methods using the Open Library API to help add standardized information to the data we already have for both tables

OL_BASE = "https://openlibrary.org/api/books"

# returns information about a book if it exists, given the ISBN
def fetch_book(isbn):
    test = requests.get(OL_BASE+"?bibkeys=ISBN:"+str(isbn)+"&jscmd=data&format=json")
    if test.status_code == 200:
        return test.json()["ISBN:"+str(isbn)]
    else:
        return None

# gets a group of books, given an array of ISBNs 
# agnostic to ISBN-10 or ISBN-13
def fetch_batch(arr):
    query = ""
    for i in arr:
        query += "ISBN:"+str(i)+","
    query = query[0:-1]
    test = requests.get(OL_BASE+"?bibkeys="+str(query)+"&jscmd=data&format=json")
    if test.status_code == 200:
        resp = test.json()
        ret = [None]*len(arr)
        for i in range(len(arr)):
            try:
                ret[i] = resp["ISBN:"+str(arr[i])]
            except KeyError:
                continue
        return ret
    else:
        return None

def fetch_batch_subsection(arr, subtitle):
    book = fetch_batch(arr)
    ret = [None]*len(book)
    for i in range(len(book)):
        try:
            sections = book[i][subtitle]
            ret[i] = [j["name"] for j in sections]
        except:
            continue
    return ret

def fetch_batch_section(arr, title):
    books = fetch_batch(arr)
    ret = [None]*len(books)
    for i in range(len(books)):
        try:
            ret[i] = books[i][title]
        except:
            continue
    return ret
    
def get_genres(isbn_arr):
    return fetch_batch_subsection(isbn_arr,"subjects")

def get_titles(isbn_arr):
    return fetch_batch_section(isbn_arr, "title")

def get_publishers(isbn_arr):
    return fetch_batch_subsection(isbn_arr,"publishers")

def get_publisher_locations(isbn_arr):
    return fetch_batch_subsection(isbn_arr,"publish_places")
     
def get_page_counts(isbn_arr):
    return fetch_batch_section(isbn_arr, "number_of_pages")

def get_publish_date(isbn_arr):
    return fetch_batch_section(isbn_arr, "publish_date")

Part 2: Data Cleaning and Management¶

New York Times - Data Cleaning¶

As mentioned before, we uploaded our code in the form of a .csv file. The original New York Times dataframe had many columns we would not need in our analysis. So before grouping our data by ISBNs, we dropped many columns and kept the following eight:

  1. Rank_List = A list of all the ranks the book has had on the bestseller list.
  2. Weeks_On_List = Total number of weeks the book has been on the bestseller list.
  3. ISBN13 = The ISBN13 of the book
  4. Publisher = The publisher of the book
  5. Title = The title of the book
  6. Author = The author of the book
  7. Contributor = The contributor of the book
  8. Dates = The dates the book was on the bestseller list
  9. Book_Type = The type of book it is (combined-print-and-e-book-nonfiction or combined-print-and-e-book-fiction)

Rank vs weeks on the list¶

Firstly, we would like to see if there is a noticable parent between the highest rank a book reached during its time on the list and how many weeks it was on the bestsellers list, and we will differentiate between fiction and non-fiction. In order to make the chart more human readable (mitigating the effects of extremes), we will take the average number of weeks for each rank for each type of book.

Part 4: Hypothesis Testing and ML¶

The Prediction of Fiction!¶

Can we use logistic regression classification to see if a book, based on the number of weeks on the bestsellers list, the highest rank it achieved, and the number of ranks it held, is fiction or not? We will use the sklearn library to build a logistic regression model as well as to create testing and training data, and eventually measure the accuracy of the model. In addition to the previous inputs, we also want to track which publisher the book is under, so we track that by creating an array with an index for every publisher in the dataset, and set the corresponding publisher index for a given book to 1, and concatenate that with the other variables.

Part 5: Discussion¶

One of the data complications that we considered involved the discrepancies between ISBN-10 and ISBN-13 values. ISBN stands for International Standard Book Number, which are purchased by publishers from an international agency that makes sure ISBNs are unique. The 10 digit and 13 digit ISBNs serve the same purpose, but the agency decided in 2007 to switch from 10 digit as the standard to 13 digits. Because the New York Times database has newer books, there are fewer books that use an isbn10 number in that dataset. While there is an algorithm that converts older ISBN-10s to ISBN-13s, the publishing sources we researched accept both. Therefore, for this project, our helper function uses both ISBN-13 and ISBN-10.

Part 6: Conclusion¶

Overview¶

According to Publishers Weekly the total number of print and e-books that were self-published in 2018 was 1.68 million. Of these 1.68 million books, only 15 get chosen as bestsellers each week. One would think that with such a large dataset, every bestseller would be vastly different from each other. However, through our analysis, we were able to see that there are some factors that play into whether your book becomes a bestseller or not. There are definite trends seen within the 2018 bestseller data such as -

  • Bestsellers tend to be in the following three genres fiction, biography, and politics
  • Bestsellers in the politics, information studies, and history genres have the highest page counts.

Based on the trends we saw, we continued to see if we could predict whether a book was fiction or not based on its bestseller data. Once we saw that this classifier was able to get a >75% accuracy, we further looked to see if we could predict the genre of the book based on bestseller information for which we got >65% accuracy.

Shortcomings¶

We were limited to using only the data for one years worth of bestseller data from the New York Times. This led to our dataset being limited (550 entries). If the scope of this project were broadened, this analysis could be spread out to a longer amount of time, e.g., 3-5 years.

While the New York Times is a highly regarded source and metric of book popularity, there is no one source that is the end all be all of what book merit or impact. For instance, the list does not take into account what books are being talked about the most on social media or which ones have the most reader generated content (e.g., reviews or fan art).

Another source that originally considered using and delved into the API was Goodreads, a social reading site owned by Amazon that allows readers to follow each other, share reviews, create lists, groups, and communities. However, after cross referencing our information with Goodreads, we found that the data available through their API did not add value within the scope of our project.

Why should you care?¶

As the world becomes more and more built on the internet, print media is becoming obsolete in many ways, as the phrase adapt or die comes into play for print newspapers, magazines, books, and other media. Readers also have more and more options for what to read, what format to consume it in, and how much to spend on it. Understanding what factors play into readers willingness to actually spend money on a book, then, is paramount for any person in the publishing industry to understand if they want to know what kind of writing to invest in, develop, and release in order to be financially viable.

Another way that bestseller lists are influential is that they serve as a way for people to understand an aspect of the national consciousness through how people are spending their time and money on one type of media.
